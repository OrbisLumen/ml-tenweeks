## 文件系统树
类 Unix 操作系统，比如 Linux，总是只有一个单一的文件系统树，不管有多少个磁盘或者存储设备连接到计算机上

## 当前工作目录
```bash

```
- 当首次登陆系统，当前工作目录就是家目录
- 普通用户身份时，家目录是唯一允许用户写入的地方

## 列出目录内容
```bash
ls      
ls -a   # 显示隐藏文件
```

## 更改当前工作目录
```bash
cd path
```
- 绝对路径 vs 相对路径

**快捷键**
```bash
cd              # 回到家目录
cd -            # 回到先前工作目录
cd ~user_name   # 回到某用户家目录
```

# 操作文件和目录

## 通配符

|通配符|意义|
|---|---|
|*|匹配任意多个字符（包括零个或一个）
|?|匹配任意一个字符（不包括零个）
|[characters]|匹配任意一个属于字符集中的字符
|[!characters]|匹配任意一个不是字符集中的字符
|\[[:class:]]|匹配任意一个属于指定字符类中的字符


|字符类|意义|
|---|---|
|[:alnum:]|匹配任意一个字母或数字
|[:alpha:]|匹配任意一个字母
|[:digit:]|匹配任意一个数字
|[:lower:]|匹配任意一个小写字母
|[:upper]|匹配任意一个大写字母

## 创建目录
```bash
mkdir directory...  # ... 表示可以某个参数可以重复
```
```bash
mkdir dir1 dir2 dir3
```
- 创建三个目录，名为dir1，dir2，dir3

## 复制文件和目录
```bash
cp item1 item2
```
- 复制单个文件或目录”item1”到文件或目录”item2”
```bash
cp item... directory
```
- 复制多个项目（文件或目录）到一个目录下

**cp选项**
|选项|意义|
|---|---|
|-a|复制文件和目录，以及它们的属性，包括所有权和权限。 通常，复本具有用户所操作文件的默认属性。
|-i|在重写已存在文件之前，提示用户确认。如果这个选项不指定， cp 命令会默认重写文件。
|-r|递归地复制目录及目录中的内容。当复制目录时， 需要这个选项（或者-a 选项）。
|-u|当把文件从一个目录复制到另一个目录时，仅复制 目标目录中不存在的文件，或者是文件内容新于目标目录中已经存在的文件。
|-v|显示翔实的命令操作信息

## 移动和命名文件夹
```bash
mv item1 item2
```
- 把文件或目录 “item1” 移动或重命名为 “item2”
```bash
mv item... directory
```
- 把一个或多个条目从一个目录移动到另一个目录中

**mv选项**
|选项|意义|
|---|---|
|-i|在重写一个已经存在的文件之前，提示用户确认信息。 如果不指定这个选项，mv 命令会默认重写文件内容。
|-u|当把文件从一个目录移动另一个目录时，只是移动不存在的文件， 或者文件内容新于目标目录相对应文件的内容。
|-v|当操作 mv 命令时，显示翔实的操作信息。

## 删除文件和目录
```bash
rm item...
```
- 删除文件和目录

**rm选项**
|选项|意义|
|---|---|
|-i|在删除已存在的文件前，提示用户确认信息。 如果不指定这个选项，rm 会默默地删除文件
|-r|递归地删除文件，这意味着，如果要删除一个目录，而此目录 又包含子目录，那么子目录也会被删除。要删除一个目录，必须指定这个选项。
|-f|忽视不存在的文件，不显示提示信息。这选项颠覆了“--interactive”选项。
|-v|在执行 rm 命令时，显示翔实的操作信息。

# 重定向

## 标准输入，输出，错误
- 标准输入 stdin    0
- 标准输出 stdout   1
- 标准错位 stderr   2

## 重定向标准输出
```bash
command > [file]
```
- 输出结果从开头开始写入
```bash
command >> [file]
```
- 输出结果添加到文件内容之后

## 重定向标准错误
```bash
command > [file] 2>&1
```
```bash
command &> file
```

## 处理不需要的输出
```bash
command 2> /dev/null
```

## 重定向标准输入
```bash
cat [file]
```
- 读取一个或多个文件，然后复制它们到标准输出

## 管道线
```bash
command | command
```
- 一个命令的标准输出管道到另一个命令的标准输入

## 过滤器
- 管道线经常用来对数据完成复杂的操作
- 有可能会把几个命令放在一起组成一个管道线
- 通常，以这种方式使用的命令被称为过滤器

## 打印相关命令
```bash
wc [file]
```
- 打印行，字和字节数
```bash
grep pattern [file]
```
- 打印匹配行
```bash
head -n [num] [file]
tail -n [num] [file]
```
- 打印文件开头/结尾部分 [num] 行
```bash
tee [file]
```
- 从 stdin 读取数据，并同时输出到 stdout 和文件
